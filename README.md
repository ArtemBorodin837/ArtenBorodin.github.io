# My Site

Персональний статичний сайт з управлінськими кейсами.

## Що це за проєкт

Це простий HTML-сайт, який використовується як публічна вітрина кейсів і професійного позиціонування.

Сайт зараз складається з двох сторінок:

- `index.html` — головна сторінка з кейсами;
- `more_cases_under_development.html` — тимчасова сторінка-заглушка для додаткових кейсів.

## Поточна структура

```text
My site/
  index.html
  more_cases_under_development.html
  README.md
```

## Як це працює

- сайт написаний на чистому HTML і CSS;
- не потребує збірки або окремого бекенду;
- може публікуватися через GitHub Pages.

## Що редагувати

### Головна сторінка

Файл:

- `index.html`

Тут змінюються:

- заголовок сайту;
- описи кейсів;
- секції з результатами;
- контактний блок;
- кнопка переходу до додаткових кейсів.

### Сторінка додаткових кейсів

Файл:

- `more_cases_under_development.html`

Тут змінюється тимчасова заглушка або майбутній контент для окремої сторінки.

## Як переглядати локально

Найпростіший спосіб:

1. відкрити папку у VS Code;
2. відкрити `index.html` у браузері.

Або, якщо використовуєте Live Server у VS Code:

1. відкрити `index.html`;
2. натиснути `Open with Live Server`.

## Публікація

Сайт можна оновлювати через GitHub Pages:

1. внести зміни у файли;
2. зробити коміт;
3. запушити зміни в GitHub;
4. дочекатися оновлення GitHub Pages.

## Ідеї для майбутнього розвитку

- винести кейси в окремі сторінки;
- покращити візуальний стиль;
- додати сильніший блок "про мене";
- додати чіткіший call to action;
- підготувати окрему сторінку контактів або форму заявки.
